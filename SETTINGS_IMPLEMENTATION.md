# Экран "Настройки" DiaBeta Mini App - Реализация

## Обзор

Полностью реализован экран настроек Telegram Mini App DiaBeta с функциями редактирования профиля, управления данными и безопасного удаления аккаунта.

---

## ✅ Реализованные функции

### 1. Профиль пользователя
- Аватар с первой буквой имени
- Отображение имени и username
- Ссылка на детальную страницу профиля (/profile)

### 2. Тип диабета
- Выпадающий список с вариантами:
  - Не указан
  - Тип 1 (TYPE_1)
  - Тип 2 (TYPE_2)
  - Гестационный (GESTATIONAL)
  - Другой (OTHER)
- Автосохранение при изменении
- PATCH /users/me

### 3. Целевой диапазон сахара
- Два поля ввода: Мин и Макс
- Тип: number с шагом 0.1
- Placeholder'ы для удобства
- Валидация на уровне backend
- Сохранение в targetGlucoseMin/targetGlucoseMax

### 4. Использование инсулина
- Toggle switch (вкл/выкл)
- Поле usesInsulin (boolean)
- Включает/отключает функции учета инсулина
- Мгновенное сохранение

### 5. Часовой пояс
- Выпадающий список российских часовых поясов:
  - Москва (UTC+3)
  - Самара (UTC+4)
  - Екатеринбург (UTC+5)
  - Омск (UTC+6)
  - Красноярск (UTC+7)
  - Иркутск (UTC+8)
  - Якутск (UTC+9)
  - Владивосток (UTC+10)
- Сохранение в timezone

### 6. Напоминания
- Ссылка на страницу /reminders
- Иконка ⏰

### 7. Подписка
- Ссылка на страницу /subscription
- Иконка 💎
- Отображение текущего тарифа (будет реализовано)

### 8. Семейный доступ
- Ссылка на страницу /family
- Иконка 👨‍👩‍👧‍👦
- Управление доступом членов семьи (будет реализовано)

### 9. Юридические документы
Раздел "Юридическая информация":
- 📄 Документы (/legal/documents)
- 🔒 Конфиденциальность (/legal/privacy)
- 📋 Условия использования (/legal/terms)

### 10. Экспорт данных
- Кнопка с иконкой 📥
- Запрос на экспорт через POST /user-data/export
- Файл придет в Telegram
- Loading state во время запроса
- Success toast после отправки

### 11. Удаление аккаунта
**Опасное действие внизу экрана:**
- Кнопка с красным текстом и иконкой 🗑️
- Модальное окно подтверждения
- Требуется ввести "УДАЛИТЬ" для подтверждения
- DELETE /users/me
- После удаления: logout + redirect на главную
- Loading state во время удаления

## 🔗 API Endpoints

### GET /users/me
Получить данные текущего пользователя

**Response:**
```json
{
  "id": "uuid",
  "telegramId": 123456789,
  "username": "john_doe",
  "firstName": "Иван",
  "lastName": "Иванов",
  "timezone": "Europe/Moscow",
  "diabetesType": "TYPE_1",
  "targetGlucoseMin": 4.0,
  "targetGlucoseMax": 7.0,
  "usesInsulin": true,
  "acceptedTermsAt": "2024-01-15T10:30:00Z",
  "acceptedPrivacyAt": "2024-01-15T10:30:00Z",
  "acceptedHealthDataConsentAt": "2024-01-15T10:30:00Z"
}
```

### PATCH /users/me
Обновить данные пользователя

**Request:**
```json
{
  "diabetesType": "TYPE_1",
  "targetGlucoseMin": 4.0,
  "targetGlucoseMax": 7.0,
  "usesInsulin": true,
  "timezone": "Europe/Moscow"
}
```

**Response:**
```json
{
  "id": "uuid",
  ...
}
```

### DELETE /users/me
Удалить аккаунт (soft delete)

**Response:**
```json
{
  "message": "Account deleted successfully"
}
```

### POST /user-data/export
Запросить экспорт данных

**Response:**
```json
{
  "message": "Export request queued. File will be sent to Telegram."
}
```

## 🚀 Как использовать

### Редактирование профиля
1. Открыть Настройки
2. Изменить тип диабета → автоматически сохраняется
3. Изменить целевой диапазон → автоматически сохраняется
4. Включить/выключить инсулин → автоматически сохраняется
5. Изменить часовой пояс → автоматически сохраняется

### Экспорт данных
1. Нажать "Экспорт данных"
2. Появится loading spinner
3. Success toast: "Запрос на экспорт отправлен"
4. Файл придет в Telegram бот

### Удаление аккаунта
1. Прокрутить вниз до "Удалить аккаунт"
2. Нажать кнопку
3. Ввести "УДАЛИТЬ" в поле подтверждения
4. Нажать "Удалить"
5. Аккаунт удаляется, происходит logout
6. Redirect на главную страницу

## 🎯 Технические детали

### State Management
- **TanStack Query** - data fetching и caching
- **useMutation** - для update/delete/export
- **queryClient.invalidateQueries** - обновление cache после изменения
- **Zustand** - authStore для logout

### Toast система
Простая реализация без внешних зависимостей:
- showToast() функция
- DOM manipulation для рендеринга
- Автоматическое скрытие через 3 секунды
- CSS анимация slide-down

### Валидация
- Backend валидирует все поля
- Frontend показывает ошибки из response
- TypeScript типы обеспечивают type safety

### Оптимизация
- Lazy loading через React.lazy (можно добавить)
- Debounce для input fields (можно добавить)
- Memoization компонентов (можно добавить React.memo)
