# Admin Panel DiaBeta

## Обзор

Admin Panel предоставляет интерфейс для управления пользователями, контентом и мониторинга системы DiaBeta.

## Стек технологий

- **Frontend**: React 18 + TypeScript
- **UI Framework**: Ant Design 5 + Refine 4
- **Data Fetching**: TanStack Query (React Query)
- **Markdown Editor**: @uiw/react-md-editor
- **Charts**: Recharts
- **Build Tool**: Vite 5

## Разделы Admin Panel

### 1. Dashboard 📊
Показывает ключевые метрики системы:
- Всего пользователей
- Новых пользователей сегодня
- Активных пользователей сегодня
- Активных пользователей за 7 дней
- Количество записей сахара
- Количество записей инсулина
- Количество созданных отчетов
- Premium пользователей
- Ошибок за последние 24 часа

### 2. Пользователи 👥
Управление пользователями:
- Поиск по имени, username, Telegram ID
- Просмотр профиля пользователя
- Управление статусом подписки
- Блокировка/разблокировка пользователей
- Soft delete по запросу

**Важно**: Медицинские записи пользователей не отображаются по умолчанию. Для доступа требуется специальное разрешение и все действия логируются в audit log.

### 3. Подписки 💳
Просмотр всех подписок:
- Список активных и истекших подписок
- Фильтрация по тарифу (FREE, BASIC, PREMIUM)
- Информация о пользователях
- Даты начала и окончания подписок

### 4. Отчеты 📄
Мониторинг генерации отчетов:
- Статус отчетов (PENDING, PROCESSING, COMPLETED, FAILED)
- Форматы (PDF, CSV, XLSX)
- Периоды отчетов
- Ссылки на скачивание файлов

### 5. База знаний 📚
Управление статьями базы знаний:
- Создание новых статей
- Редактирование существующих
- Markdown редактор с preview
- Управление статусами (draft/published/archived)
- Категоризация статей
- Удаление статей

### 6. Ошибки ⚠️
Мониторинг ошибок системы:
- Список ошибок из audit logs
- Фильтрация по модулю
- Фильтрация по статусу обработки
- Информация о пользователях
- IP адреса и user agents

## Backend API Endpoints

Все admin endpoints требуют JWT аутентификацию:

```
GET    /admin/dashboard          - Статистика dashboard
GET    /admin/users              - Список пользователей
GET    /admin/users/:id          - Профиль пользователя
PATCH  /admin/users/:id          - Обновление пользователя
GET    /admin/subscriptions      - Список подписок
GET    /admin/reports            - Список отчетов
GET    /admin/knowledge/articles - Список статей (включая draft)
POST   /admin/knowledge/articles - Создание статьи
PATCH  /admin/knowledge/articles/:id - Обновление статьи
DELETE /admin/knowledge/articles/:id - Удаление статьи
GET    /admin/errors             - Список ошибок
```

## Безопасность

### Аутентификация
- Отдельная admin auth система
- Роли: admin, super_admin
- JWT токены с ограниченным временем жизни
- Refresh token механизм

### Авторизация
- Проверка прав доступа на каждом endpoint
- Role-based access control (RBAC)
- Audit log всех действий администраторов

### Защита данных
- Медицинские данные скрыты по умолчанию
- Требуется явное разрешение для доступа к чувствительным данным
- Все действия логируются
- GDPR compliance

## Установка и запуск

### 1. Установка зависимостей

```bash
cd packages/admin
npm install
```

### 2. Настройка окружения

Создайте `.env` файл:

```env
VITE_API_URL=http://localhost:3001/api
```

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно на http://localhost:3000

### 4. Сборка для продакшена

```bash
npm run build
```

## Best Practices

### Для администраторов

1. **Безопасность**
   - Никогда не делитесь учетными данными
   - Используйте сложные пароли
   - Регулярно меняйте пароли
   - Логаут после завершения работы

2. **Работа с пользователями**
   - Перед блокировкой проверьте историю
   - Документируйте причины блокировки
   - Уведомляйте пользователей об изменениях

3. **База знаний**
   - Проверяйте орфографию перед публикацией
   - Используйте понятные slug
   - Categorize статьи правильно
   - Регулярно обновляйте контент

4. **Мониторинг ошибок**
   - Регулярно проверяйте список ошибок
   -及时处理 критические ошибки
   - Документируйте решения
   - Отслеживайте тренды

## Troubleshooting

### Не могу войти
- Проверьте правильность credentials
- Убедитесь, что backend запущен
- Проверьте консоль браузера на ошибки

### Данные не загружаются
- Проверьте网络连接 к backend
- Убедитесь, что токен действителен
- Проверьте CORS настройки

### Markdown редактор не работает
- Убедитесь, что все зависимости установлены
- Проверьте версию @uiw/react-md-editor
- Очистите кэш браузера
