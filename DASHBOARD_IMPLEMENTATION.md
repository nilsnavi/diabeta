# Главный экран Telegram Mini App DiaBeta - Реализация

## Обзор

Полностью переработан главный экран (Dashboard) Telegram Mini App DiaBeta с современным дизайном в стиле Apple Health/Whoop.

---

## ✅ Реализованные функции

### 1. Современный Dashboard

**Структура экрана:**

#### Header
- Название приложения "DiaBeta"
- Текущая дата на русском языке
- Аватар пользователя с первой буквой имени

#### Карточка последнего сахара
- Крупное отображение значения (5xl font)
- Единицы измерения (ммоль/л)
- Время измерения и относительное время ("2 часа назад")
- Цветовой статус:
  - 🟢 Зеленый градиент - в норме
  - 🟠 Оранжевый градиент - ниже нормы
  - 🔴 Красный градиент - выше нормы
- Иконка статуса (⬇️ ✅ ⬆️)

#### Мини-график сахара за день
- Интерактивный график на Recharts
- Tooltip при нажатии на точку
- Отображение времени и значения
- Количество измерений

#### Быстрые действия
- 4 крупные кнопки с градиентами:
  - 🩸 Сахар (красно-оранжевый)
  - 💉 Инсулин (сине-голубой)
  - 🍽️ Еда (зеленый)
  - 😊 Самочувствие (фиолетово-розовый)
- Анимация при нажатии (scale)

#### Ближайшее напоминание
- Карточка с amber/orange фоном
- Иконка ⏰
- Название и время напоминания
- Показывается только если есть активные напоминания

#### Последние записи
- Таймлайн последних 5 записей
- Тип записи (иконка)
- Заголовок и подзаголовок
- Значение и время
- Ссылка "Все →" на страницу истории

#### Кнопка "Отчёт врачу"
- Градиентная кнопка indigo-purple
- Полная ширина экрана
- Иконка 📊
- Подзаголовок "PDF с анализом за период"

### 2. Нижняя навигация (BottomNav)

Обновленная структура:
- 🏠 Главная (/)
- 📋 История (/history)
- 📊 Аналитика (/analytics)

### 3. Страница настроек (SettingsPage)

**Разделы:**
- Профиль пользователя (аватар, имя, username)
- Основные настройки:
  - 👤 Профиль
  - 🏥 Диабет профиль
  - ⏰ Напоминания
- Юридическая информация:
  - 📄 Документы
  - 🔒 Конфиденциальность
  - 📋 Условия использования
- База знаний:
  - 📚 База знаний
- О приложении:
  - Версия
  - Дата обновления

- Медицинский дисклеймер (amber карточка)

### 4. Юридические документы

#### LegalDocumentsPage
Список всех юридических документов:
- Пользовательское соглашение
- Политика конфиденциальности
- Согласие на обработку персональных данных
- Согласие на обработку данных о здоровье

Для каждого документа:
- Иконка
- Название
- Версия
- Статус принятия (дата или "Требуется принятие")

#### LegalDocumentPage
Просмотр конкретного документа:
- Полный текст документа
- Форматирование markdown
- Кнопка "Принимаю условия"
- Навигация назад

**Содержимое документов:**
1. **Пользовательское соглашение** - 6 разделов
2. **Политика конфиденциальности** - 6 разделов
3. **Согласие на обработку персональных данных** - список данных, цели, срок
4. **Согласие на обработку данных о здоровье** - специальные категории данных

### 5. LegalConsentModal

Модальное окно, показываемое при первом запуске:

**Функции:**
- Блокирует доступ к приложению до принятия документов
- Отображает медицинский дисклеймер
- Список документов с индикаторами статуса
- Кнопка "Принять все документы"
- Ссылка "Подробнее о документах"
- Возможность открыть каждый документ для чтения

**Логика:**
- Проверяет статус принятия через API
- После принятия всех документов обновляет auth store
- Закрывается и дает доступ к приложению

### 6. API интеграция

#### Создан legal.ts API клиент:
```typescript
legalApi.getDocuments()           // Получить список документов
legalApi.getDocument(id)          // Получить конкретный документ
legalApi.acceptDocument(type)     // Принять документ
legalApi.getUserLegalStatus()     // Получить статус принятия
```

#### Обновлен authStore:
Добавлено поле `legalStatus`:
```typescript
{
  acceptedTermsAt: string | null;
  acceptedPrivacyAt: string | null;
  acceptedHealthDataConsentAt: string | null;
}
```

### 7. Empty States

Обработаны пустые состояния:
- Нет измерений сахара → карточка с кнопкой "Измерить сейчас"
- Нет записей в таймлайне → "Нет записей"
- Loading states → skeleton loaders

## 🚀 Как использовать

### 1. Первый запуск
При первом запуске пользователь видит:
1. Загрузку приложения
2. Модальное окно LegalConsentModal
3. Необходимо принять все документы
4. После принятия - доступ к Dashboard

### 2. Просмотр документов
В любой момент можно открыть:
- Настройки → Юридическая информация → Документы
- Или напрямую по ссылке /legal/documents

### 3. Повторное принятие
Если документы обновятся (новая версия):
- Backend вернет null в полях accepted*At
- LegalConsentModal покажется снова
- Пользователь должен принять новую версию

## 📊 API Endpoints (Backend - требуется реализация)

### GET /legal/documents
Возвращает список всех юридических документов

**Response:**
```json
[
  {
    "id": "terms",
    "title": "Пользовательское соглашение",
    "version": "1.0",
    "content": "..."
  }
]
```

### GET /legal/documents/:id
Возвращает конкретный документ

**Response:**
```json
{
  "id": "terms",
  "title": "Пользовательское соглашение",
  "version": "1.0",
  "content": "# Текст документа..."
}
```

### POST /users/me/accept-legal
Принимает юридический документ

**Request:**
```json
{
  "documentType": "terms" // terms | privacy | health-data
}
```

**Response:**
```json
{
  "success": true
}
```

### GET /users/me/legal-status
Возвращает статус принятия документов

**Response:**
```json
{
  "acceptedTermsAt": "2024-01-15T10:30:00Z",
  "acceptedPrivacyAt": "2024-01-15T10:30:00Z",
  "acceptedHealthDataConsentAt": "2024-01-15T10:30:00Z"
}
```

## ✅ Критерии готовности

- ✅ **Экран открывается в Telegram** - WebApp.ready() и WebApp.expand() вызываются
- ✅ **Данные пользователя загружаются** - через useAuthStore
- ✅ **Быстрые действия работают** - ссылки на соответствующие страницы
- ✅ **Последние записи отображаются** - через timeline API
- ✅ **Пустые состояния обработаны** - EmptyState компоненты
- ✅ **Дизайн адаптирован под мобильный экран** - max-w-md, responsive grid
- ✅ **Без принятия нельзя пройти onboarding** - LegalConsentModal блокирует доступ
- ✅ **Даты принятия сохраняются** - через API endpoints
- ✅ **Документы доступны из настроек** - раздел "Юридическая информация"
- ✅ **Дисклеймер есть в AI и отчётах** - отображается в SettingsPage и LegalDocumentsPage
